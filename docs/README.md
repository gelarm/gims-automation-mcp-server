# Инструкции для LLM агентов

Этот каталог содержит файлы с инструкциями для различных LLM-агентов по работе с данным MCP сервером.

## Доступные файлы

- **CLAUDE.md** - Инструкции для Claude Code и Claude.ai
- **AGENTS.md** - Инструкции для универсальных агентских систем
- **QWEN.md** - Инструкции для Qwen Code и Qwen-based агентов

> **Примечание:** Все три файла содержат идентичные инструкции, но названы по-разному для удобства интеграции с различными системами.

## Установка и настройка

### Claude Code

Для использования с Claude Code скопируйте содержимое docs/CLAUDE.md в файл .claude/CLAUDE.md в каталоге проекта разработки автоматизаций:
```bash
mkdir -p .claude
cat docs/CLAUDE.md >> .claude/CLAUDE.md
```

> **Примечание:** При первом запуске Claude Code в проекте используйте команду `/init` - Claude Code автоматически создаст файл `.claude/CLAUDE.md` с описанием вашего проекта. Вы можете дополнить его инструкциями из `docs/CLAUDE.md`.

### Cursor

Для использования с Cursor IDE копируйте содержимое docs/AGENTS.md в файл .cursorrules в каталоге проекта разработки автоматизаций:
```bash
cat docs/AGENTS.md >> .cursorrules
```

### Qwen Code

Для использования с Qwen Code скопируйте содержимое docs/QWEN.md в файл QWEN.md в каталоге проекта разработки автоматизаций:
```bash
cat docs/QWEN.md >> QWEN.md
```

> **Примечание:** При первом запуске Qweb Code в проекте используйте команду `/init` - Qwen Code автоматически создаст файл `QWEN.md` с описанием вашего проекта. Вы можете дополнить его инструкциями из `docs/QWEN.md`.

### Другие агентские системы

Для других систем (Aider, Continue, Codeium, и т.д.):

1. Используйте файл `AGENTS.md` как универсальные инструкции

2. Добавьте содержимое в соответствующий файл конфигурации вашей системы:
   - **Aider**: `.aider/instructions.md`
   - **Continue**: `.continue/config.json` (в поле `systemMessage`)
   - **Codeium**: `.codeium/instructions.md`

## Структура инструкций

Файлы инструкций содержат:

- Описание доступных MCP инструментов/функций
- Примеры использования сервера
- Рекомендации по работе с API
- Лучшие практики интеграции

## Проверка установки

Чтобы убедиться, что инструкции правильно загружены, спросите вашего агента:

```
"Какие MCP инструменты доступны в этом проекте?"
```

Агент должен перечислить инструменты, описанные в файлах инструкций.

## Поддержка

Если у вас возникли проблемы с интеграцией:

1. Проверьте, что файл инструкций находится в правильном месте для вашей системы
2. Убедитесь, что MCP сервер запущен и доступен
3. Перезапустите вашу агентскую систему
4. Создайте issue в репозитории с описанием проблемы
